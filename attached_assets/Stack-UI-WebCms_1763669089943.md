# Stack UI para Web y CMS

## 1. Librería de componentes base

**Librería principal:** `shadcn/ui`

- Colección de componentes accesibles (inputs, cards, tabs, dialogs, etc.) construidos con **React + Tailwind + Radix**.
- En vez de ser un paquete cerrado, copias el código de los componentes a tu repo: encaja muy bien con tu enfoque de monolito modular y `shared/ui`.

**Complemento de bloques/plantillas sobre shadcn:**

**Launch UI Components**

- Conjunto de bloques y templates **gratuitos y open source** construidos *exactamente* con **React + shadcn/ui + Tailwind**.
- Incluye heroes, secciones de pricing, FAQs, call-to-actions, etc., ya maquetados con buenas prácticas responsive → ideal para decirle al Agent:
    
    > “Usa el bloque Hero de Launch UI adaptado a web/sections/HeroKinesis.tsx con estos props…”
    > 

**Ventaja clave para ti:**

- Mantienes **un solo “lenguaje visual”** entre CMS y Web (shadcn + Tailwind).
- Minimiza fricción para el Agent: no le obligas a mezclar muchas librerías diferentes.

## 2. Template visual específico para la escuela de danza

**Template base (referencia de layout y estética):**

**Serene Yoga Landing – Tailwind CSS template (TailFlux)**

Por qué lo elijo como base para Kinesis:

- Está pensado para **estudios** (yoga/meditación), muy cercano a una escuela de danza:
    - **Hero** con claim principal + CTA.
    - Sección de **clases/programas**.
    - Sección de **instructores/equipo**.
    - **Testimonios**.
    - **Horario** / schedule con días y horas.
- Es 100% **Tailwind**, lo que facilita portar los layouts a React + Vite y envolverlos en componentes de sección propios (`<ProgramsSection>`, `<ScheduleSection>`, etc.).

La idea **no** es usar su código tal cual, sino:

1. Copiar el **esqueleto de secciones y estructura responsive** como referencia visual.
2. Implementar esas secciones usando **componentes de shadcn + bloques de Launch UI** en React, dentro de tu `web/`.
